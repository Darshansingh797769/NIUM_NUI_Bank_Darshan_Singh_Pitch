Security X‑Api‑Key, TLS, per‑request x-request-id, idempotent writes, secrets in vault.
Data Persist IDs (clientHashId, customerHashId, walletHashId, systemReferenceNumber); pass‑through docs to NIUM.
Compliance Region‑aware onboarding; RFIs; sanction screening by NIUM.
Treasury Prefund guardrails, balances, statements, alerts.

Core Flow ▸ API Mapping (Corporate Only)

  1. Prefund Liquidity → POST /client/{clientHashId}/prefund · GET /client/{clientHashId}/balances
  2. Corporate KYB → POST /v1/client/{clientHashId}/corporate → handle GET/POST /corporate/rfi → POST /customer/{customerHashId}/corporate/documents
  3. Hosted Onboarding (optional) → POST /v1/client/{clientHashId}/applications
  4. Wallets & VRAs → POST /customer/{customerHashId}/wallet · /wallet/{walletHashId}/paymentId(s)
  5. FX → GET /v2/exchangeRate or POST /quotes → POST /.../conversions
  6. Beneficiary + Payout → POST /v2/.../beneficiaries → /beneficiaries/validate → POST /.../remittance → GET /.../audit
  7. Ops & Reco → GET /client/{clientHashId}/transactions and statements + webhooks.

FX — Bulk, Per‑Transaction, Lock & Hold

Get Quote (Lock Rate)

POST • Quotes
curl -X POST 'https://gateway.nium.com/api/v1/client/{clientHashId}/quotes' \
 -H 'Content-Type: application/json' -H 'X-Api-Key: <API_KEY>' \
 -d '{"sourceCurrency":"USD","destinationCurrency":"INR","amount":100000,"lockPeriod":"24H"}'

Use returned quoteId within validity window for execution.

Execute Conversion

POST • Conversions
curl -X POST 'https://gateway.nium.com/api/v1/client/{clientHashId}/customer/{customerHashId}/wallet/{walletHashId}/conversions' \
 -H 'Content-Type: application/json' -H 'X-Api-Key: <API_KEY>' \
 -d '{"quoteId":"<QUOTE_ID>","settlementType":"LOCKED"}'

# Bulk FX: repeat per currency pair/amount; schedule EOD/T+1 where supported

How NIUM Solves Niu Bank’s Concerns — Q&A

Concern 1: High Transaction Fees

Solution: NIUM routes payments via its partner network and local clearing systems, cutting out intermediaries → lowering FX margins & bank charges.

Concern 2: Sluggish Processing Times

Solution: With direct‑to‑rail integrations, NIUM processes payments instantly in certain corridors (e.g., UPI in India, Faster Payments in UK) and within T+0/T+1 globally.

Concern 3: Lack of Clarity & Visibility

Solution: NIUM APIs provide real‑time payment status, predictable FX, and complete transaction reporting for corporate treasury teams.

Concern 4: Integration & Client Experience
  • Corporate Onboarding APIs → faster, compliant customer setup.
  • Wallet & Virtual Account APIs → hold, fund, and disburse across currencies.
  • Payout APIs → seamless global payouts with a single integration.
